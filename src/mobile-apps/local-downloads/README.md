# Workaround for local blob downloads

## Problem

When testers started trying out the mobile apps, they found that several document downloads were not working (excel, pdf).

Pdf, excel or csv files are generated by the frontend and then a local download is forced. It works for the web app, but not for the mobile apps, the app does not crash but the download does not happen.

Examples:

- Should download an excel file: Dashboard > Gráfica de consumo > ver consumo > scroll al final > exportar datos de consumo > exportar datos de la gráfica > exportar

- Should download a pdf file: Suministros > entrar en un suministro > gestionar mi suministro > Mis certificados > seleccionar tipo Consumos > rellenar fechas > Descargar


## Solution

As a workaround, right after the file should be downloaded, if platform is mobile app, the file contents is saved in the device app directory and then opened with the default app or a native dialog to choose the app.

This has been applied to approximately 36 points.

Approx 25 excel or csv local downloads via XLSX library (https://sheetjs.com/)
```diff
        XLSX.writeFile(wb, fileName)

+       // XLSX.writeFile will attempt to force a client-side download, works for web,
+       // for mobile apps blob downloads do not work ATM, code below is an alternative, (hover createFileAndOpenIt for jsdoc).
+       if (isMobileApp()) createFileAndOpenIt({ fileName, contentAsBase64: XLSX.write(wb, { type: 'base64' }) })
```

Approx 5 pdf local downloads
```diff
    const pdfUrl = window.URL.createObjectURL(blob)
    const tempLink = document.createElement('a')
    tempLink.href = pdfUrl
    const fileName = 'Acuerdo entre titulares.pdf'
    tempLink.setAttribute('download', fileName)
    tempLink.click()

+   // above code will attempt to force a client-side download, works for web,
+   // for mobile apps blob downloads do not work ATM, code below is an alternative, (hover createFileAndOpenIt for jsdoc).
+   if (isMobileApp()) await createFileAndOpenIt({ fileName, contentAsBlob: blob })
```

For this solution two new capacitor plugins have been added, to be able to save files in the device app directory and to open them with the default app or a native dialog to choose the app.
```
    "@capacitor/filesystem": "^4.1.4",
    "@capawesome-team/capacitor-file-opener": "^0.0.2",
```

We call this a workaround because the best solution would be blob downloads supported by Capacitor, we have read community discussions and it is a known issue, it is being worked on, but it is not ready yet.
